# CLAUDE.md — dashboard-frontend

Operating guide for agents working in this repo. Human onboarding is in `README.md`.

## What this is

A **React dashboard SPA template** — the frontend half of a full-stack base. It pairs with
**`dashboard-backend`** (a Hono + SQLite API in a sibling repo): this app owns the UI, auth
session handling, and the dashboard/projects views; the backend owns persistence and is the
security boundary. They're **separate repos** and meant as a **base for future projects** —
keep it conventional and copy-pasteable. The structure follows the blog article
_"Structuring a Frontend Repo in 2026"_.

## Stack (locked — don't relitigate without the owner)

React 19 + TypeScript + **Vite**. **TanStack Router** (typed, file-based, auto code-split).
**SWR** for server state, **Zustand** for client state. **Zod** + a `Result`-type client at
the API boundary. **React Hook Form** + Zod for forms. **Tailwind v4** + **shadcn/ui**.
**Recharts / Sonner / TanStack Table / Lucide / date-fns**. **Biome** lint+format,
**Vitest + Testing Library + MSW** for tests.

## The two rules that keep this navigable

1. **One owner per kind of state.** SWR owns anything from the network (projects, dashboard
   stats, the session). Zustand owns UI/ephemeral state (sidebar, the in-memory access
   token). **Never copy server data into Zustand** or fake client state through a data hook.
2. **Colocate by domain; reusability earns its way up the tree.** A domain owns its types,
   raw API calls, and hooks together under `api/<domain>/`. One-off UI lives in a route's
   `-components/`; promote to `components/` only when a second consumer appears.

## Layout

- `api/api.ts` — the **single** client. Auth header injection, 401→refresh→retry, and error
  normalization live here and nowhere else. Returns a `Result<T>`, never throws.
- `api/result.ts` — `Result`, `ApiError`, and `parseResponse` (Zod validation at the edge).
- `api/<domain>/` — `types.ts` (Zod schemas + inferred types) · `<domain>.ts` (raw fns) ·
  `use<Domain>*.ts` (SWR hooks — the cache boundary; **components call hooks, never raw fns**).
- `components/ui/` shadcn primitives (**immutable** — compose, never fork) → `components/blocks/`
  (compound, reusable) → `routes/.../-components/` (route-local).
- `lib/stores/` Zustand · `lib/hooks/` · `lib/utils.ts` (`cn`).
- `routes/` file-based; `_authed.tsx` is the pathless auth guard, `context/auth.tsx` bootstraps
  the session and wires the client's refresh callback. `config.ts` is the only reader of env.

## Commands

- `pnpm dev` — Vite (HMR; the router plugin regenerates `routeTree.gen.ts`).
- `pnpm build` — `tsr generate && tsc --noEmit && vite build`.
- `pnpm typecheck` — `tsr generate && tsc --noEmit` (generate first so the route tree exists).
- `pnpm check` / `pnpm check:fix` — Biome.
- `pnpm test` / `pnpm test:watch` — Vitest.

## Conventions & gotchas

- **`@/` is the path alias for `src/`** (tsconfig `paths` + Vite `resolve.alias`).
- **The route tree is generated** (`src/routeTree.gen.ts`, gitignored). It's regenerated by
  the Vite plugin in dev and by `tsr generate` in the typecheck/build/test scripts — so a
  standalone `tsc` needs `tsr generate` first (that's why the scripts prepend it). Config in
  `tsr.config.json` mirrors the Vite plugin (autoCodeSplitting on, `-` ignore prefix).
- **Hooks are named `use<Domain><Entity>`** and a `null` SWR key disables a fetch
  (`useProject(undefined)`). Global SWR config (no refetch-on-focus, dedup, no retry) is in
  `main.tsx`.
- **RHF `formState` is a Proxy** — destructure the fields you read (`const { errors, isSubmitting } = …`)
  or they won't subscribe. Controlled shadcn inputs (Select) go through `<Controller>`.
- **Validate, don't assume.** Add a Zod schema in `types.ts` and parse with `parseResponse`;
  `as Type` is banned at the wire boundary. Mirror the backend's contracts here.
- **Auth = in-memory access token + httpOnly refresh cookie.** Don't put the token in
  localStorage. The backend is the security boundary — the FE guards routes for UX, not security.
- **Biome doesn't lint CSS** here (`styles.css` uses Tailwind v4 at-rules the parser rejects);
  Tailwind owns styling. Don't edit shadcn primitives to fix one call site — fix the call site.

## Testing

Vitest + happy-dom; mock the network with **MSW** at the API boundary (`src/test/msw/`), with
`onUnhandledRequest: "error"`. Render with the providers helper in `src/test/test-utils.tsx`.
Unit-test the pure seams (`Result`/Zod mapping, Zustand stores via `getState()`, query-key
builders); integration-test SWR hooks against MSW (cover the schema-drift and error-envelope
paths); render shared components. Co-locate `*.test.ts(x)` next to the code. Add tests for the
behavior you touch — auth/session, the API client, and money/stat math are the high-blast-radius areas.

## Commits

**Commit/push only when the owner asks.** This is a template under active authoring — make and
verify changes (typecheck + test + `biome check` + a `vite build`), but leave git to the owner.
